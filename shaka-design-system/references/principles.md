# 设计原则 · Principles

**不是品味，是可执行的约束。**

每条原则都有**为什么**（理由）和**如何应用**（具体场景），避免变成空洞的口号。

---

## 原则 1：约束式设计 —— 枚举优于自由

### 规则
**不允许用户自定义 hex 色值 / 字体 / 布局比例**。只能从预设中选。

### 为什么
- 99% 的用户配色会翻车（饱和度过高、对比度不够、配色不协调）
- 一旦允许自由配，设计系统就失去"同一视觉语言"的意义
- AI agent 自由发挥 = 每次输出不一致，用户记不住品牌感

### 如何应用
- 主题：只能从 `themes.md` 的 5 套选
- 字体：只能从 `fonts.md` 的三层（衬线 / 无衬线 / 等宽）选
- 组件：只能从 `components.md` 枚举的块选，不允许现造

### 委婉拒绝用户自定义的话术模板

> "我来推荐一套匹配你内容的主题——你这次是偏科技感（靛蓝瓷）还是偏文艺（牛皮纸）？我把 5 套主题截图给你选。自定义颜色的话大概率会和其他元素打架，没好处。"

## 原则 2：规则文档化 —— 设计不是品味

### 规则
**所有设计决策必须能被 agent 查询到**。不能"靠感觉"，必须有规则文档。

### 为什么
- 你不在场时 agent 需要自己做决定，靠感觉 = 翻车
- 规则沉淀成文档 = 经验可复用，不依赖个人
- 记忆的 Why 帮 agent 在边界情况下做判断

### 如何应用
- 每条规则配 **Why + How to apply**
- "血泪经验" → 立刻写进 `checklist.md`，不靠下次记得
- 约束有争议时，加 "决策日志"：`2026-04-XX 因为 YYY 选择 ZZZ`

## 原则 3：节奏优于元素 —— 整体呼吸感

### 规则
**8+ 页/段的内容必须有深浅对比和视觉呼吸**，不允许 3 页/段连续同色调。

### 为什么
- 单页/单段再漂亮，连续 10 页同色调 = 审美疲劳
- 杂志感来自"hero 震撼 → 内容详述 → hero 再震撼"的节奏
- 读者的视觉需要"休息节点"

### 如何应用（PPT）
- 每 2-3 页切换一次 `data-theme="light"` 和 `dark`
- 至少 1 个 hero 页（封面/章节过渡/结尾金句）
- 连续 3 页同色调 → 必须加 1 个过渡页

### 如何应用（公众号长图文）
- 5 段式结构交替"浅段 / 深段 / 浅段 / 深段 / 浅段"
- 或"头条重块 → 要闻轻块 → 动态重块 → 短评卡片 → 预告轻块"
- 段落之间加视觉分隔（分隔线 / 留白 / emoji 符号）

### 如何应用（小红书）
- 多图封面：首图 hero，中间图内容，末图 CTA
- 单图封面：前景 / 中景 / 背景三层不同灰度

## 原则 4：踩坑沉淀 —— 血泪经验进 checklist

### 规则
**任何一次出错 / 不如预期 / 被反馈"丑" / 客户抱怨，必须进 `checklist.md`**，不能"下次注意"就算了。

### 为什么
- 人会忘记，agent 不会。规则进 checklist 后永不遗漏
- 同一错误犯两次 = 工程化失败
- 外部客户只看你最差的一次输出

### 如何应用
每条 checklist 格式：
```
- [P0/P1/P2/P3] 错误描述 —— 原因 —— 如何避免
```

P0 = 会丑到不能发布
P1 = 会被客户吐槽
P2 = 有更好做法但不致命
P3 = 细节完美度

## 原则 5：共享 vs 特化 —— 单一真相源

### 规则
**所有跨媒介可复用的规则进 `shaka-design-system`，业务特化规则进下游 skill**。

### 为什么
- 避免同一个主题色在 3 个 skill 里改 3 遍
- 下游 skill 升级时不影响其他 skill
- 新增输出形态（小红书 / 邮件 / 抖音）只需加一个下游 skill

### 判断"共享 vs 特化"的分界线

| 内容 | 归属 | 理由 |
|------|------|------|
| 5 套主题色板 | 共享（`shaka-design-system/themes.md`） | 跨媒介通用 |
| 字体层级（衬线/无衬线/等宽） | 共享（`shaka-design-system/fonts.md`） | 排版语言一致 |
| callout / stat / pillar 组件 | 共享（`shaka-design-system/components.md`） | HTML 片段跨媒介可复用 |
| 公众号 5 段式结构 | 特化（`ai-tech-wechat-writer/layouts.md`） | 只公众号业务需要 |
| PPT 10 版式 | 特化（`guizang-ppt-skill/layouts.md`） | 只 PPT 需要 |
| 横滑 JS 导航 | 特化（`guizang-ppt-skill/assets/`） | 只 PPT 需要 |
| 公众号编辑器限制（无 JS、无外链 SVG） | 特化（`ai-tech-wechat-writer/constraints.md`） | 公众号平台约束 |

## 原则 6：跨媒介一致性 —— 看得出是一家

### 规则
**Shaka 的公众号、PPT、小红书、邮件应该一眼看出是"同一个人"的内容**。

### 为什么
- 个人品牌建设需要视觉记忆
- 读者在不同平台遇到你，应该有"哦这是他"的识别
- 设计系统是品牌的外化

### 如何应用
- 字体栈同源（Noto Serif SC + Noto Sans SC + IBM Plex Mono）
- 主题色板同源（5 套主题跨媒介一致）
- 组件命名同源（`callout` 在公众号、PPT、小红书都叫 callout）
- 装饰元素同源（分隔线、kicker 等宽标签、章节编号罗马数字）
