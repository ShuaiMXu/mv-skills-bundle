# Seedance 2.0 vs Sora vs Veo · 对比与选型

> 来源：PDF《Seedance 2.0 使用手册》第九章

## 一览表

| 工具 | 核心优势 | 适用场景 | 成本 | LibTV 支持 |
|------|---------|---------|------|-----------|
| **Seedance 2.0** | 角色一致性强、可控性高、参考视频支持 | 商业广告、短剧、MV | 中等 | ✅ 默认模型 |
| **Sora** | 生成质量高、叙事能力强 | 电影短片、创意实验 | 高 | 部分支持 |
| **Veo** | 速度快、操作简单 | 社交媒体短视频 | 低 | 不支持 |
| **Kling 3.0 / O3** | 高质量写实 + 电影感 | 高质量定制视频 | 中高 | ✅ 可显式要求 |
| **Wan 2.6** | 通用视频生成 | 一般创作 | 中 | 部分支持 |

## LibTV 默认策略

根据 `libtv-skill` v0.0.3 的更新日志（2026-04-02）：

- **默认视频生成模型**：Seedance 2.0
- **支持显式切换**：可在提示词中要求「使用 kling O3」或「使用 kling 3.0」
- **仿真人人像**：Seedance 2.0 默认支持

## 选型决策树

当用户提出视频生成需求时，参考以下逻辑判断是否需要提示 libtv 后端切换模型：

```
用户需求
  │
  ├── 需要角色贯穿多镜头 → Seedance 2.0（默认，无需切换）
  ├── 需要真人写实人像 → Seedance 2.0（默认，无需切换）
  ├── 需要电商产品展示 → Seedance 2.0（默认，无需切换）
  ├── 需要短剧/MV/TVC → Seedance 2.0（默认，无需切换）
  │
  ├── 追求极致画质质感（电影短片） → 建议用户显式要求 Kling O3
  ├── 需要 Sora 级叙事能力 → 建议用户显式要求 Sora（如果 LibTV 支持）
  │
  └── 只是随手玩玩 / 快速测试 → Seedance 2.0（默认，先低分辨率）
```

## 何时建议用户显式指定非默认模型？

**场景 1：用户明确提到「质感」「电影感」「高质量写实」**
> 建议话术：「这个需求对画质要求很高，默认 Seedance 2.0 也能做，但如果你想要更极致的电影感，可以在描述里加一句『使用 Kling O3 生成』让后端切到 Kling。」

**场景 2：用户描述的是抽象艺术或超现实场景**
> 建议话术：「这种偏实验性的画面，Sora 的叙事感可能更强。要不要在描述里指定模型？或者先用 Seedance 默认版看效果？」

**场景 3：用户就是想快速出片 / 不在意细节**
> 直接默认 Seedance 2.0，不做多余干涉。

## 核心结论

**Seedance 2.0 是目前商业创作性价比最高的选择**，尤其适合需要**角色一致性**和**多场景控制**的项目。

对于 shaka 这种以漫剧 / 短剧 / 商业内容为主的工作流，**默认选 Seedance 2.0 就够用**。除非有特殊画质需求，否则不需要主动提示切换。

## 参考价格参考（PDF 未披露具体数字，以下为行业参考）

- Seedance 2.0：约 0.1-0.5 元 / 秒（视分辨率而定）
- Sora：约 1-3 元 / 秒
- Veo：约 0.05-0.2 元 / 秒
- Kling 3.0/O3：约 0.2-0.8 元 / 秒
